# Language style guide for this project

This file is OPTIONAL and language-specific. Drop in any rules that apply ONLY when translating into this language for this project — for example:

- Voice/register exceptions ("more conversational than the global brand voice")
- Specific terminology choices ("translate `Dashboard` as `Tableau de bord`, never `Bord de bord`")
- Formatting quirks for this audience (numerals, dates, units of measure)
- Cultural or legal notes specific to the target market

Anything written here STACKS on top of:
1. The project-level `styleGuide` field in projects.json (general brand voice)
2. The built-in locale rules in src/config/style-guide.ts (formal pronouns, etc.)

Leave the file empty (or delete the file entirely) if no per-language overrides are needed.
